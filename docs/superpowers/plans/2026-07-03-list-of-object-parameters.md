# List-of-Object Parameters Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 6 fields across 5 typed resources (`variable_matomoconfiguration.custom_dimensions`/`custom_data`, `tag_matomo.custom_dimensions`, `variable_etrackerconfiguration.custom_dimensions`, `tag_googleanalytics4event.event_parameters`, `tag_googleconsentmodev2.consent_types`) that are currently modeled as flat `list(string)` but need Matomo's real list-of-named-objects wire shape, plus a 7th field (`variable_matomoconfiguration.domains`) that needs only its wire encoding corrected. Add equivalent generic-resource support via a new `parameter_list` block shared across `matomo_tagmanager_tag`/`trigger`/`variable`.

**Architecture:** A new third `ParamValue` shape (`ListOfObjects`) in `internal/matomo/formencoding.go` is the foundation everything else builds on - wire encoding/decoding for "a list of rows, each with named string fields." Each of the 6 typed resources gets a real nested `schema.ListNestedBlock` replacing its flat `schema.ListAttribute`, with row-shape-specific (not generic) Go conversion code, hand-edited directly into the already-generated files (this plan does not modify `tools/gen` - see Global Constraints for why). The generic resources gain one new shared block type, `parameter_list`, using nested `row`/`item` sub-blocks.

**Tech Stack:** Go, terraform-plugin-framework (`schema.ListNestedBlock`).

## Global Constraints

- This plan hand-edits 5 files marked `// Code generated by tools/gen. DO NOT
  EDIT.` (`generated_variable_matomoconfiguration.go`, `generated_tag_matomo.go`,
  `generated_variable_etrackerconfiguration.go`,
  `generated_tag_googleanalytics4event.go`, `generated_tag_googleconsentmodev2.go`).
  This is a deliberate, scoped exception: extending `tools/gen` itself to
  auto-detect this parameter shape (Matomo's `UI_CONTROL_MULTI_TUPLE`) needs
  its own investigation and design (see the design spec's section 5.5) and is
  out of scope for this plan. If `tools/gen` is later extended to regenerate
  these 5 files, whoever does that must reproduce this exact fix (or add
  equivalent override-table entries) or a regeneration will silently revert
  it - flag this prominently in that future work, don't let it get lost.
- The 6 real list-of-object fields have per-field row shapes, not one fixed
  convention - verified against Matomo's live PHP source, not assumed:
  `customDimensions` -> `{index, value}`, `customData` -> `{name, value}`,
  `eventParameters` -> `{parameter, value}`, `consentTypes` ->
  `{consent_type, consent_state}` (renamed to `{type, state}` per this
  project's decision - Terraform-facing only, the wire keys stay
  `consent_type`/`consent_state`).
- `domains` is `TYPE_ARRAY` + `UI_CONTROL_MULTI_TUPLE` in Matomo too, but each
  row has exactly one key (`domain`) - it stays a flat `list(string)` in
  schema/model/HCL; only its wire encoding changes (see Task 2).
- This environment cannot run acceptance tests requiring a live Matomo
  instance (no Docker daemon) or `make docs` (network-blocked). Defer
  `TestAcc*`/`TF_ACC=1` runs and the docs drift-check to CI, matching this
  project's established convention. Unit tests for the core data model (Task
  1) run locally, since they need no live Matomo.

- [ ] **Step 3: Implement `ParamValue.ListOfObjects`**

lines 21-24) to:

```go
// ParamValue is one entry in a Tag/Trigger/Variable's flat "parameters"
// map: a plain scalar, a list, or a list of objects. Matomo's own
// dispatcher builds PHP arrays for array-typed parameters from the raw
// query string's native name[]=x / name[i][key]=x convention, not from a
// JSON-encoded string value - confirmed against a live instance (a
// JSON-encoded array string was rejected outright with "<Field>: ... has to
// be an array"). List is non-nil (even if zero-length) exactly when a value
// represents a flat list of scalars; ListOfObjects is non-nil exactly when
// it represents a list of rows, each with its own named string fields
// (Matomo's UI_CONTROL_MULTI_TUPLE parameters - e.g. MatomoConfiguration's
// customDimensions is a list of {index, value} rows, confirmed against
// Matomo's own MatomoConfigurationVariable.php source, not merely a flat
// list of strings the way ListParam alone could represent).
type ParamValue struct {
	Scalar        string
	List          []string
	ListOfObjects []map[string]string
}
```

Add two new functions after the existing `ListParam`/`IsList`:

```go
// ListOfObjectsParam builds a list-of-objects-typed ParamValue.
func ListOfObjectsParam(rows []map[string]string) ParamValue { return ParamValue{ListOfObjects: rows} }

// IsListOfObjects reports whether v represents a list-of-objects-typed value.
func (v ParamValue) IsListOfObjects() bool { return v.ListOfObjects != nil }

// WrapSingleKeyParam builds a ListOfObjects-typed ParamValue from a flat
// list of strings, wrapping each one as a single-key row - for parameters
// like MatomoConfiguration's domains, which Matomo's PHP source declares as
// UI_CONTROL_MULTI_TUPLE (so it needs the same [i][key]=value wire shape as
// any other list-of-objects parameter) but whose rows only ever have one
// key, so there's no pairing information a nested Terraform block would add
// over a flat list(string) - this keeps the Terraform-facing schema flat
// while still sending Matomo the shape it expects.
func WrapSingleKeyParam(key string, items []string) ParamValue {
	rows := make([]map[string]string, len(items))
	for i, item := range items {
		rows[i] = map[string]string{key: item}
	}
	return ListOfObjectsParam(rows)
}
```

- [ ] **Step 4: Implement `addParamsMap`'s `ListOfObjects` case**

```go
func addParamsMap(v url.Values, name string, m ParamsMap) {
	for key, val := range m {
		if val.IsListOfObjects() {
			for i, row := range val.ListOfObjects {
				for rowKey, rowValue := range row {
					v.Set(fmt.Sprintf("%s[%s][%d][%s]", name, key, i, rowKey), rowValue)
				}
			}
			continue
		}
		if val.IsList() {
			for _, item := range val.List {
				v.Add(fmt.Sprintf("%s[%s][]", name, key), item)
			}
			continue
		}
		v.Set(fmt.Sprintf("%s[%s]", name, key), val.Scalar)
	}
}
```

- [ ] **Step 5: Implement `decodeParamValue`'s array-of-objects case**

Change `decodeParamValue` (currently lines 76-99) to detect an array of
objects before falling back to array-of-strings:

```go
func decodeParamValue(raw json.RawMessage) ParamValue {
	var s string
	if err := json.Unmarshal(raw, &s); err == nil {
		return ScalarParam(s)
	}
	var objArr []map[string]string
	if err := json.Unmarshal(raw, &objArr); err == nil && len(objArr) > 0 {
		return ListOfObjectsParam(objArr)
	}
	var arr []string
	if err := json.Unmarshal(raw, &arr); err == nil {
		return ListParam(arr)
	}
	var b bool
	if err := json.Unmarshal(raw, &b); err == nil {
		if b {
			return ScalarParam("1")
		}
		return ScalarParam("0")
	}
	var f float64
	if err := json.Unmarshal(raw, &f); err == nil {
		return ScalarParam(strconv.FormatFloat(f, 'f', -1, 64))
	}
	return ScalarParam("")
}
```

(The `len(objArr) > 0` guard matters: an empty JSON array `[]` unmarshals
successfully into both `[]map[string]string{}` and `[]string{}` - without
this guard, an empty list-of-strings parameter would be misdecoded as an
empty list-of-objects. This also means a real empty *object-list* parameter
falls through to the `arr []string` branch instead, decoding as an empty
`ListParam` rather than an empty `ListOfObjectsParam` - harmless, since
`FromParams` implementations in Tasks 3-7 treat "key absent" and "key present
but empty" identically regardless of which empty-list shape is returned.)

// parameterListModel is the generic resources' equivalent of a typed
// resource's real nested block (e.g. custom_dimension{index,value}) - since
// the generic resources have no per-type knowledge of a parameter's real
// row shape, each row is a generic list of key/value items rather than
// named attributes.
type parameterListModel struct {
	Name types.String        `tfsdk:"name"`
	Row  []parameterRowModel `tfsdk:"row"`
}

type parameterRowModel struct {
	Item []parameterItemModel `tfsdk:"item"`
}

type parameterItemModel struct {
	Key   types.String `tfsdk:"key"`
	Value types.String `tfsdk:"value"`
}

// parameterListsToMap builds the ListOfObjects-shaped entries of a
// Tag/Trigger/Variable's "parameters" map from the generic parameter_list
// blocks - the counterpart to parametersToMap, which only builds scalar
// entries from the plain parameter{} blocks.
func parameterListsToMap(lists []parameterListModel) matomo.ParamsMap {
	m := make(matomo.ParamsMap, len(lists))
	for _, pl := range lists {
		rows := make([]map[string]string, len(pl.Row))
		for i, row := range pl.Row {
			r := make(map[string]string, len(row.Item))
			for _, item := range row.Item {
				r[item.Key.ValueString()] = item.Value.ValueString()
			}
			rows[i] = r
		}
		m[pl.Name.ValueString()] = matomo.ListOfObjectsParam(rows)
	}
	return m
}

// parameterListsFromAPI extracts every ListOfObjects-shaped entry of params
// back into parameter_list blocks, sorting each row's items by key
// alphabetically for deterministic state (Matomo's own response has no
// inherent key order within a row, since it's decoded from a JSON object).
func parameterListsFromAPI(params matomo.ParamsMap) []parameterListModel {
	var lists []parameterListModel
	names := make([]string, 0, len(params))
	for name := range params {
		names = append(names, name)
	}
	sort.Strings(names)
	for _, name := range names {
		val := params[name]
		if !val.IsListOfObjects() {
			continue
		}
		rows := make([]parameterRowModel, len(val.ListOfObjects))
		for i, row := range val.ListOfObjects {
			keys := make([]string, 0, len(row))
			for k := range row {
				keys = append(keys, k)
			}
			sort.Strings(keys)
			items := make([]parameterItemModel, len(keys))
			for j, k := range keys {
				items[j] = parameterItemModel{Key: types.StringValue(k), Value: types.StringValue(row[k])}
			}
			rows[i] = parameterRowModel{Item: items}
		}
		lists = append(lists, parameterListModel{Name: types.StringValue(name), Row: rows})
	}
	return lists
}
